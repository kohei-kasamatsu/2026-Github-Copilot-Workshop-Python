# 実装する必要のある機能一覧

## 1. 目的

本ドキュメントは、ポモドーロタイマー Web アプリケーションを実装するにあたって必要となる機能を整理したものである。

対象は [1.pomodoro](1.pomodoro) 配下で構築する Flask + HTML/CSS/JavaScript ベースのアプリケーションであり、[architecture.md](architecture.md) で定義したアーキテクチャ方針に沿って分類している。

## 2. MVP で実装する機能

以下は、まず最初に実装すべき機能である。

### 2.1 画面表示機能

- ポモドーロタイマー画面を表示する
- タイトルを表示する
- 現在のモードを表示する
- 残り時間を中央に表示する
- 円形プログレスを表示する
- 開始ボタンを表示する
- 一時停止状態を視覚的に区別できるようにする
- リセットボタンを表示する
- 今日の進捗カードを表示する
- 完了回数を表示する
- 累積集中時間を表示する
- モバイルとデスクトップで破綻しないレイアウトにする

### 2.2 タイマー操作機能

- 作業タイマーを開始する
- 実行中タイマーを一時停止する
- 一時停止したタイマーを再開する
- 現在モードの初期時間へリセットする
- タイマー終了時に残り時間を 00:00 で止める

### 2.3 タイマー進行機能

- 残り時間を 1 秒単位で更新表示する
- 現在時刻と終了予定時刻の差から残り時間を再計算する
- 経過率に応じて円形プログレスを更新する
- 作業モード、短休憩モード、長休憩モードを扱う
- 作業完了後に休憩モードへ自動遷移する
- 一定回数ごとの作業完了後に長休憩へ遷移する
- 休憩完了後に次の作業モードへ戻る

### 2.4 今日の進捗機能

- 当日の完了ポモドーロ回数を記録する
- 当日の累積集中時間を記録する
- 画面上に今日の進捗を反映する
- 日付が変わった場合は新しい日付で集計を開始する

### 2.5 状態保存と復元機能

- タイマー状態を localStorage に保存する
- 実行中か停止中かを保存する
- 現在モードを保存する
- 残り時間または終了予定時刻を保存する
- 今日の進捗情報を保存する
- ページ再読み込み時に保存済み状態を復元する
- 復元時に期限切れのタイマー状態を正しく解決する

### 2.6 Flask 基本機能

- トップページを配信する
- HTML テンプレートを描画する
- CSS と JavaScript の静的ファイルを配信する
- 作業時間、短休憩時間、長休憩時間などの初期設定を管理する

## 3. 実装時に必要な内部ロジック機能

以下はユーザー向け機能を支える内部ロジックであり、実装対象として明確に含めるべきものである。

### 3.1 タイマー計算ロジック

- 総時間と終了予定時刻から残り秒数を求める
- 総時間と残り秒数から進捗率を求める
- 残り時間が 0 秒未満にならないように補正する
- 表示用に秒数を mm:ss 形式へ変換する

### 3.2 モード遷移ロジック

- 作業完了時の次モードを判定する
- 長休憩に入る条件を判定する
- 休憩完了後の戻り先を判定する
- モード切り替え時に初期時間を再設定する

### 3.3 状態管理ロジック

- 開始イベントを適用する
- 一時停止イベントを適用する
- 再開イベントを適用する
- リセットイベントを適用する
- TICK イベントを適用する
- COMPLETE イベントを適用する
- 状態変更後に保存処理へ渡す

### 3.4 進捗集計ロジック

- 作業完了時に完了回数を加算する
- 作業完了時に集中時間を加算する
- 日付キーごとに進捗を保持する
- 日付変更時に当日の進捗を切り替える

### 3.5 ストレージアダプタ機能

- localStorage から安全に読み込む
- 保存形式を JSON として統一する
- 不正データや欠損データを扱えるようにする
- 保存キーを一元管理する

## 4. ユニットテストしやすくするために実装すべき機能

以下は品質確保のため、実装段階から分離しておくべき機能である。

### 4.1 pure function 化する対象

- 残り時間計算関数
- 進捗率計算関数
- 次モード判定関数
- 長休憩判定関数
- 日次集計更新関数
- 表示用時刻フォーマット関数

### 4.2 依存分離する対象

- DOM 操作を画面描画専用関数へ分離する
- localStorage アクセスをストレージアダプタへ分離する
- 現在時刻取得を差し替え可能にする
- Flask のルート処理と業務ロジックを分離する

### 4.3 テスト対象として想定する機能

- 残り時間計算の境界値確認
- 進捗率計算の境界値確認
- 作業完了時の短休憩遷移確認
- 規定回数到達時の長休憩遷移確認
- 日付切り替え時の集計初期化確認
- ページ復元時の状態整合性確認
- Flask のトップページ応答確認

## 5. 将来的に追加しやすい拡張機能

以下は MVP には含めないが、将来追加しやすいように考慮しておく機能である。

### 5.1 設定機能

- 作業時間の変更
- 短休憩時間の変更
- 長休憩時間の変更
- 長休憩に入る回数の変更

### 5.2 通知機能

- タイマー完了時の通知表示
- 通知音の再生
- ブラウザ通知の送信

### 5.3 履歴と統計機能

- 日別履歴の保存
- 週次統計の表示
- 月次統計の表示
- 完了セッション一覧の表示

### 5.4 サーバーサイド拡張機能

- 設定取得 API
- 今日の統計取得 API
- セッション保存 API
- SQLite による永続化
- 複数端末間のデータ共有
- 認証機能の追加

## 6. 優先順位

実装優先度は以下の順が妥当である。

1. 画面表示機能
2. タイマー操作機能
3. タイマー進行機能
4. 今日の進捗機能
5. 状態保存と復元機能
6. ユニットテストしやすい内部構造の分離
7. 将来拡張向け API と永続化

## 7. 結論

本アプリケーションで実装すべき中核機能は、タイマー表示、開始停止操作、モード遷移、今日の進捗集計、状態保存と復元である。加えて、ユニットテストしやすさを確保するために、計算ロジック、状態管理、UI、ストレージアクセスを分離して実装する必要がある。

これにより、まずはシンプルな MVP を成立させつつ、後から設定機能、通知、統計、サーバー永続化へ無理なく拡張できる。
